6.3.1 Study Notes

📌 Why Do We Need Session Management?

  • Prevents users from sharing credentials and abusing services.
  • Ensures controlled access, e.g., limiting to one active session per user.
  • Helps track and manage user sessions to maintain security.

📌 JWT & Session Management

  • JWT (JSON Web Token) is stateless – typically, we don't manage sessions in a database.
  • To restrict multiple active sessions, we need to store session information in a database.
  • We track:
    • Session ID
    • Refresh Token
    • User ID
    • Last Used Time

🔥 How Session Management Works?

Scenario: Restricting to 1 Active Session

1️⃣ User logs in → Creates a session in DB.

2️⃣ If another login attempt occurs:

  • Check session limit (e.g., 1 session allowed).
  • If session exists, delete the previous session.
  • Create new session → Issue fresh JWT.

3️⃣ When JWT expires:

  • Verify refresh token.
  • If session exists → Issue new access token.
  • If session does not exist → Force new login.

3️⃣ Implementing Session Service

✅ Generating a New Session


@Service
public class SessionService {
    private final SessionRepository sessionRepository;
    private final int SESSION_LIMIT = 2;  // Max active sessions

    public void generateNewSession(User user, String refreshToken) {
        List<Session> userSessions = sessionRepository.findByUser(user);

        if (userSessions.size() >= SESSION_LIMIT) {
            // Remove least recently used session
            userSessions.sort(Comparator.comparing(Session::getLastUsedAt));
            sessionRepository.delete(userSessions.get(0));
        }

        // Create a new session
        Session newSession = new Session();
        newSession.setUser(user);
        newSession.setRefreshToken(refreshToken);
        newSession.setLastUsedAt(LocalDateTime.now());

        sessionRepository.save(newSession);
    }
}

4️⃣ Validating Session Before Refresh


public void validateSession(String refreshToken) {
    Session session = sessionRepository.findByRefreshToken(refreshToken)
            .orElseThrow(() -> new SessionAuthenticationException("Invalid Session"));

    // Update session timestamp
    session.setLastUsedAt(LocalDateTime.now());
    sessionRepository.save(session);
}

5️⃣ Integrating with Authentication Flow

🌟 Updating Login API


public AuthResponse login(LoginRequest request) {
    User user = userService.authenticate(request.getUsername(), request.getPassword());

    String accessToken = jwtService.generateAccessToken(user);
    String refreshToken = jwtService.generateRefreshToken(user);

    // Create a new session
    sessionService.generateNewSession(user, refreshToken);

    return new AuthResponse(accessToken, refreshToken);
}

🌟 Updating Token Refresh API


public AuthResponse refreshToken(String refreshToken) {
    // Validate token and session
    jwtService.validateRefreshToken(refreshToken);
    sessionService.validateSession(refreshToken);

    // Generate new access token
    String newAccessToken = jwtService.generateAccessToken(user);
    return new AuthResponse(newAccessToken, refreshToken);
}

🎯 Testing the Flow

1️⃣ User Logs In (Session Created)

✅ Check sessions table → New session entry added.

2️⃣ Another Login Attempt

✅ Old session deleted, new session created.

3️⃣ Using Refresh Token

✅ If session exists, new access token is issued.

❌ If session is deleted, user must re-login.

4️⃣ Trying to Exceed Session Limit

✅ Oldest session is deleted when new session is created.

📌 Homework (Enhancements)

🔹 Create Logout API → Delete session on logout.

🔹 Dynamic Session Limits → Different users can have different session limits.

🔹 Store Session Count in Subscription Entity instead of hardcoding limit.

⚡ Summary (Quick Revision for Interviews)

1️⃣ Why Manage Sessions? → Prevents credential sharing, limits active logins.

2️⃣ How? → Store session details in DB & track refresh tokens.

3️⃣ Key Steps:

  • Login: Create a new session, delete old session if limit exceeded.
  • Refresh Token: Check session before issuing new token.
  • Session Expiry: No session = No refresh token allowed.

4️⃣ Database Tables: Store session ID, refresh token, user ID, last used time.

5️⃣ Enhancements: Logout API, dynamic session limits.
