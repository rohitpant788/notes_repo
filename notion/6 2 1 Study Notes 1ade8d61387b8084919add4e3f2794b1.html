6.2.1 Study Notes

üîπ Introduction to OAuth 2 Client Authentication

OAuth 2.0 Client Authentication allows applications to authenticate users using third-party authorization servers such as Google, Facebook, GitHub, etc. instead of relying on traditional username and password authentication.

üîπ Traditional Authentication Flow (Email & Password)

  • User enters email and password.
  • The password is encrypted and stored in the database.
  • The application verifies credentials and authenticates the user.

üîπ OAuth 2.0 Authentication Flow

  • Instead of managing user credentials, the app delegates authentication to an external provider (Google, GitHub, etc.).
  • The provider handles authentication and returns a token with user details.
  • The application then uses this token to authenticate the user and provide access.

üîπ Setting Up OAuth 2 Client in Spring Boot

1Ô∏è‚É£ Adding Dependencies

To use OAuth 2 Client, add the following dependency in pom.xml:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>

üìå Important: Refresh Maven after adding the dependency to download required packages.

2Ô∏è‚É£ Configuring OAuth 2 Client

Configuration is done in the application.properties or application.yaml file.

Using application.properties

spring.security.oauth2.client.registration.google.client-id=YOUR_CLIENT_ID
spring.security.oauth2.client.registration.google.client-secret=YOUR_CLIENT_SECRET

Using application.yaml (Recommended)

spring:
  security:
    oauth2:
      client:
        registration:
          google:
            client-id: YOUR_CLIENT_ID
            client-secret: YOUR_CLIENT_SECRET

üõë Why YAML?

  • Avoids redundant key-value pairs.
  • Easier to read and manage.

üîπ Updating WebSecurityConfig for OAuth2 Authentication

After adding OAuth2 dependencies and configuring the client ID and secret, we need to update WebSecurityConfig to enable OAuth2 authentication.

1Ô∏è‚É£ Modify WebSecurityConfig

Update the security configuration to allow OAuth2 authentication:

@Configuration
@EnableWebSecurity
@AllArgsConstructor
public class WebSecurityConfig {

    private final JwtAuthFilter jwtAuthFilter;
    private final OAuth2SuccessHandler oAuth2SuccessHandler;
    
    private static final String[] publicRoutes = {
            "/error", "/auth/**", "/home.html"
    };

    @Bean
    SecurityFilterChain securityFilterChain(HttpSecurity httpSecurity) throws Exception {
        httpSecurity
                .authorizeHttpRequests(
                        auth -> auth
                                .requestMatchers(publicRoutes).permitAll()
                                //.requestMatchers(HttpMethod.GET, "/posts/**").permitAll()
                                .requestMatchers(HttpMethod.POST, "/posts/**")
                                    .hasAnyRole(ADMIN.name(),CREATOR.name())
                                .anyRequest().authenticated())
                .csrf(AbstractHttpConfigurer::disable)
                .sessionManagement(sessionConfig -> sessionConfig
                        .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
                .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
                .oauth2Login(oauth2Config -> oauth2Config
                        .failureUrl("/login?error=true")
                        .successHandler(oAuth2SuccessHandler)
                );
        ;

        return httpSecurity.build();
    }



    @Bean
    AuthenticationManager authenticationManager(AuthenticationConfiguration authenticationConfiguration) throws Exception {
        return authenticationConfiguration.getAuthenticationManager();
    }
}

2Ô∏è‚É£ Explanation

  • @EnableWebSecurity: Enables Spring Security.
  • securityFilterChain(HttpSecurity http):
    • Disables CSRF protection (optional).
    • Allows unrestricted access to / and /home.
    • Requires authentication for all other requests.
    • Configures OAuth2 login:
      • Uses a custom success handler (OAuth2SuccessHandler).
      • Redirects to /login?error=true in case of failure.

‚úÖ Why is this needed?

Without this configuration, Spring Security won't handle OAuth2 authentication requests properly.

üîπ Setting Up Google OAuth 2.0 Credentials

Step 1: Google Cloud Console Setup

  1. Go to Google Cloud Console.
  1. Create a new project (or use an existing one).
  1. Navigate to APIs & Services > Credentials.
  1. Click Create Credentials > OAuth 2.0 Client ID.

Step 2: Configure OAuth Consent Screen

  • Choose External (if users outside your organization will log in).
  • Enter App Name, Support Email, and Developer Email.
  • Define OAuth Scopes:
    • ‚úÖ Non-sensitive scopes: Profile, Email.
    • ‚ùå Sensitive scopes: Gmail, Calendar (require Google approval).

Step 3: Create OAuth Client ID

  • Choose Web Application as the Application Type.
  • Set Authorized Redirect URIs:
    http://localhost:8080/login/oauth2/code/google

    (Spring Security automatically handles this URL.)

  • Click Create ‚Üí Copy Client ID & Secret.

üîπ Implementing OAuth2 Success Handler

1Ô∏è‚É£ Create OAuth2SuccessHandler

@Component
@RequiredArgsConstructor
@Slf4j
public class OAuth2SuccessHandler extends SimpleUrlAuthenticationSuccessHandler {
    private final UserService userService;
    private final JwtService jwtService;


    @Value("${deploy.env}")
    private String deployEnv;

    @Override
    public void onAuthenticationSuccess(HttpServletRequest request, HttpServletResponse response,
                                         Authentication authentication) throws IOException, ServletException {

        OAuth2AuthenticationToken token = (OAuth2AuthenticationToken) authentication;
        DefaultOAuth2User oAuth2User = (DefaultOAuth2User) token.getPrincipal();

        String email = oAuth2User.getAttribute("email");

        User user = userService.getUserByEmail(email);

        if(user == null) {
            User newUser = User.builder()
                    .name(oAuth2User.getAttribute("name"))
                    .email(email)
                    .build();

            user = userService.save(newUser);
        }

        String accessToken = jwtService.generateAccessToken(user);
        String refreshToken = jwtService.generateRefreshToken(user);

        Cookie cookie = new Cookie("refreshToken", refreshToken);
        cookie.setHttpOnly(true);
        cookie.setSecure("production".equals(deployEnv));
        response.addCookie(cookie);

        String frontEndUrl = "http://localhost:8080/home.html?token="+accessToken;

        response.sendRedirect(frontEndUrl);
    }

}

üìå Key Responsibilities:

  • Retrieves user details.
  • Checks if user exists; if not, creates a new user.
  • Generates JWT tokens.
  • Saves the refresh token in a cookie.
  • Redirects user to frontend with access token.

üîπ Supporting Facebook & GitHub Authentication

To add Facebook & GitHub login, simply extend application.yaml:

spring:
  security:
    oauth2:
      client:
        registration:
          google:
            client-id: YOUR_GOOGLE_CLIENT_ID
            client-secret: YOUR_GOOGLE_CLIENT_SECRET
          github:
            client-id: YOUR_GITHUB_CLIENT_ID
            client-secret: YOUR_GITHUB_CLIENT_SECRET
          facebook:
            client-id: YOUR_FACEBOOK_CLIENT_ID
            client-secret: YOUR_FACEBOOK_CLIENT_SECRET

‚úÖ Result:

On /login, you will now see Google, GitHub, and Facebook login buttons!

üîπ Summary: Quick Interview Revision

‚úÖ OAuth 2 Client Authentication Flow:

  1. User clicks Login with Google.
  1. App redirects to Google Authorization Server.
  1. Google asks for credentials & consent.
  1. Google returns AUTH_CODE.
  1. App exchanges AUTH_CODE for Access & Refresh Tokens.
  1. App authenticates user and sets RT in cookies.
  1. App redirects user to frontend with Access Token.

‚úÖ Spring Security OAuth 2 Setup Steps:

  1. Add spring-boot-starter-oauth2-client dependency.
  1. Configure OAuth Client ID & Secret in application.yaml.
  1. Set up WebSecurityConfig to use OAuth2 login.
  1. Implement OAuth2SuccessHandler to process tokens.
  1. Store refresh tokens in cookies and access token in response.
  1. Redirect user to frontend.

‚úÖ OAuth2SuccessHandler Responsibilities:

  • Extracts user details from OAuth2 token.
  • Creates a new user if not present.
  • Generates JWT access & refresh tokens.
  • Saves refresh token in a cookie.
  • Redirects user to frontend with access token.
