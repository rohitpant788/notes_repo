1.7 Maven Build Tool

üîπ Introduction to Maven

Maven is a build automation and project management tool for Java applications. It plays a crucial role in dependency management, build automation, and deployment.

üì¢ Key Takeaways:

‚úÖ Maven is not specific to Spring Boot; it works for any Java project.

‚úÖ It helps manage dependencies and automates project builds.

‚úÖ Applications can be packaged, tested, and deployed using Maven.

‚úÖ Maven Central Repository hosts various dependencies that can be pulled into projects.

üîπ Maven as a Chef üçΩÔ∏è (Analogy)

Think of Maven as a chef who wants to cook a dish.

  • Recipe (pom.xml) ‚Üí Defines what is needed.
  • Ingredients (Dependencies) ‚Üí Pulled from Maven repositories.
  • Final Dish (Packaged Application - JAR/WAR) ‚Üí Built and ready for deployment.

Maven orchestrates everything: it fetches dependencies, compiles code, runs tests, and packages the project for deployment.

üîπ What is pom.xml?

POM = Project Object Model (An XML file that contains project configurations).

üì¢ What does pom.xml define?

  • Java version
  • Dependencies (Libraries used in the project)
  • Plugins
  • Repositories (like Maven Central)
  • Project metadata

Developers specify dependencies in pom.xml, and Maven handles everything else.

üîπ Maven's Role in Dependency Management

‚úÖ Automatic dependency resolution ‚Äì No need to manually download JARs.

‚úÖ Transitive dependencies ‚Äì If dependency A requires dependency B, Maven fetches both.

‚úÖ Versioning & Conflict Resolution ‚Äì Handles different versions of dependencies.

üìå Example: Spring Boot starter dependencies (spring-boot-starter-web, etc.) are all managed in pom.xml via Maven.

üîπ Maven Build Automation üîÑ

Maven automates the entire build process, including:

‚úÖ Compilation

‚úÖ Testing

‚úÖ Packaging

‚úÖ Deployment

üí° Maven Build Lifecycle:

Each phase in Maven's build lifecycle is called a goal.

üì¢ Maven Build Phases (Goals)

PhaseDescription
validateChecks if project structure and dependencies are correct.
compileCompiles Java source files.
testRuns unit tests using JUnit/TestNG.
packageCreates a JAR/WAR file.
installInstalls JAR/WAR into the local repository.
deployDeploys packaged application to a remote repository.
