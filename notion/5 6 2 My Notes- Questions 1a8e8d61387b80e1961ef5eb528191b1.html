5.6.2 My Notes- Questions

Question:

In a signup API, the user enters name, email, and password, and we use BCrypt to encode the password before storing it in the database.

  • If someone has DBA (Database Administrator) access, can they simply use BCrypt to decrypt the stored password?
  • Is BCrypt encryption reversible?
  • How does BCrypt protect passwords, and why is it considered secure?
Answer:

1Ô∏è‚É£ Can a DBA Decrypt a Stored BCrypt Password?

‚ùå No, a DBA (or anyone) CANNOT simply decrypt a BCrypt password because BCrypt is NOT reversible.

üìå Why?

  • BCrypt is a one-way hashing algorithm, not encryption.
  • Unlike encryption (which is reversible with a key), hashing is designed to be irreversible.
  • Even if a DBA has access to the database, they cannot retrieve the original password from a hashed value.

‚úÖ Example: Storing a Password Using BCrypt

String hashedPassword = encoder.encode("mySecurePassword");
System.out.println(hashedPassword);

2Ô∏è‚É£ How Does BCrypt Protect Passwords?

‚úÖ BCrypt is designed to resist brute-force attacks using two key techniques:

FeatureExplanation
SaltEach password hash is generated with a random salt, making each hash unique. Even if two users have the same password, their hashes will be different.
Adaptive Work Factor (Cost Factor)BCrypt allows setting a work factor (e.g., 10, 12, 14), making it computationally expensive to brute-force.

ReasonExplanation

4Ô∏è‚É£ How Does Password Verification Work?

When a user logs in:

  1. The user enters their password.
  1. The backend hashes the input password with BCrypt.
  1. The new hash is compared to the stored hash.

üìå Example: Verifying a Password in Java

BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();

// Store this hash in the database during signup
String storedHash = encoder.encode("mySecurePassword");

// User enters password at login
String inputPassword = "mySecurePassword";

// Check if the entered password matches the stored hash
boolean isMatch = encoder.matches(inputPassword, storedHash);
System.out.println("Password Match: " + isMatch);

‚úÖ If the password is correct, matches() will return true.

üö® Notice:

  • We never decrypt the stored password.
  • Instead, we rehash the input and compare the hashes.

5Ô∏è‚É£ Can a DBA or Attacker Crack the Password?

If an attacker gets the database, they cannot decrypt the hashes, but they could:

  • Try to brute-force it (but the cost factor makes this slow).
  • Use dictionary attacks (if users have weak passwords).
  • Use precomputed tables (but salt makes this ineffective).

‚úÖ How to Protect Against This?

  • Use a strong work factor (e.g., BCryptPasswordEncoder(12)).
  • Force strong passwords (e.g., 12+ characters, mix of letters, numbers, symbols).
  • Rate-limit login attempts to slow down brute-force attacks.
